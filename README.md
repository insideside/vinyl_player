# insideside music

insideside music — персональный музыкальный веб-плеер с визуализацией в стиле виниловой пластинки. Работает как локальное приложение или как сервер в домашней сети, доступный с любого устройства. Поддерживает импорт треков из VK, Spotify, Apple Music, Яндекс.Музыки и SoundCloud, автоматический поиск метаданных и обложек, создание плейлистов, офлайн-кэширование и установку как PWA на iPhone/iPad.

---

## Локально или сервер?

**Локальный режим** (по умолчанию) — плеер запускается на `http://127.0.0.1:7656` и доступен только на вашем компьютере. Подходит для персонального использования.

**Серверный режим (LAN/WAN)** — плеер становится доступен другим устройствам в локальной сети (LAN) или через интернет (WAN). Все подключённые устройства используют одну библиотеку, но каждый пользователь имеет собственный аккаунт и набор каталогов.

## Установка

### Требования

- Python 3.7+
- macOS / Linux / Windows
- `openssl` (для HTTPS в режиме LAN — обычно предустановлен)
- `cloudflared` (опционально, только для WAN-режима через Cloudflare Tunnel)

### Установка зависимостей

```bash
pip install mutagen httpx vkpymusic musicbrainzngs
```

| Пакет | Назначение |
|-------|------------|
| `mutagen` | Чтение и запись метаданных аудиофайлов (ID3, FLAC, M4A) |
| `httpx` | HTTP-клиент для запросов к внешним API |
| `vkpymusic` | Интеграция с VK Music (поиск и скачивание треков) |
| `musicbrainzngs` | Поиск метаданных в базе MusicBrainz |

### Поддерживаемые форматы

MP3, FLAC, M4A, OGG, WAV, AAC, OPUS.

### Запуск

```bash
python3 vinyl_player.py
```

Плеер автоматически откроется в браузере по адресу `http://127.0.0.1:7656`.

Для запуска сразу в LAN-режиме:

```bash
python3 vinyl_player.py --public
```

### Установка Cloudflare Tunnel (опционально, для WAN)

**macOS:**
```bash
brew install cloudflared
```

**Linux (Debian/Ubuntu):**
```bash
curl -fsSL https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg >/dev/null
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/cloudflared.list
sudo apt update && sudo apt install cloudflared
```

**Windows:**
```powershell
winget install Cloudflare.cloudflared
```

## Учётные записи

### Первый запуск — создание администратора

При первом запуске вы увидите форму создания аккаунта администратора. Укажите логин, пароль и корневую папку для музыки (по умолчанию `~/VinylMusic`). Администратор создаётся один раз — это главный аккаунт с полными правами.

### Управление пользователями

Администратор может добавлять новых пользователей через панель управления (кнопка с иконкой шестерёнки в верхней панели). Для каждого пользователя задаются логин, пароль и роль.

**Роли пользователей:**

| Возможность | Админ | Пользователь | Демо |
|-------------|:-----:|:------------:|:----:|
| Слушать музыку | ✓ | ✓ | ✓ |
| Добавлять каталоги | ✓ | ✓* | — |
| Скачивать из VK / импорт | ✓ | ✓ | — |
| Поиск метаданных | ✓ | ✓ | — |
| Перемещать / переименовывать треки | ✓ | ✓ | — |
| Управлять плейлистами | ✓ | ✓ | — |
| Включать LAN/WAN | ✓ | — | — |
| Управлять пользователями | ✓ | — | — |
| Скачивать ZIP-архив каталога | ✓ | — | — |

\* Пользователь может добавлять каталоги только внутри корневой папки музыки.

**Демо-аккаунт** — режим «только слушать». Демо-пользователь видит только специальную папку `_demo` и не может изменять библиотеку. Удобно для гостевого доступа.

### Безопасность

- Пароли хранятся в виде хэша PBKDF2-SHA256 (260 000 итераций, индивидуальная соль для каждого пользователя)
- Сессии — HttpOnly cookie с атрибутом SameSite=Strict и 30-дневным сроком действия
- Защита от перебора паролей: максимум 5 попыток на IP и на пользователя за 5 минут, плюс глобальный лимит 20 попыток
- Файлы конфигурации, сессий и сертификатов доступны только владельцу (chmod 600)
- Защита от path traversal на всех файловых эндпоинтах

## Каталоги

### Корневая папка музыки

При первом запуске задаётся корневая папка (по умолчанию `~/VinylMusic`). Администратор может изменить её в панели управления. Внутри неё для каждого пользователя автоматически создаётся подпапка с его логином.

### Работа с каталогами

- **Добавить каталог** — нажмите кнопку «+» в верхней части панели. Откроется файловый браузер для выбора папки на диске. После выбора каталог появится в выпадающем списке
- **Удалить каталог** — нажмите кнопку «−» рядом с выбранным каталогом. Удаляется только привязка к аккаунту, файлы на диске остаются нетронутыми
- **Переключение** — выберите нужный каталог из выпадающего списка. Плеер просканирует папку и покажет все найденные аудиофайлы

### Разграничение прав между пользователями и каталогами

- **Администратор** видит все каталоги всех пользователей и может добавлять любые папки на диске без ограничений
- **Пользователь** видит только свои каталоги и может добавлять папки только внутри корневой папки музыки (защита MUSIC_ROOT sandbox)
- **Демо** видит только папку `_demo`, без возможности добавления

Администратор может назначить конкретные каталоги каждому пользователю через панель управления → список пользователей → настройка папок.

## Серверный режим

### LAN — доступ в локальной сети

1. Включите тумблер **LAN** в верхней панели (виден только администратору на локальной машине)
2. Сервер автоматически сгенерирует самоподписанный HTTPS-сертификат и перезапустится на `0.0.0.0`
3. В информационной строке появятся LAN-адреса (например, `https://192.168.0.157:7656`)
4. Откройте этот адрес на другом устройстве в той же Wi-Fi сети
5. При первом подключении браузер покажет предупреждение о самоподписанном сертификате — нажмите «Дополнительно» → «Перейти на сайт»

При выключении тумблера LAN сервер возвращается в локальный режим. Состояние LAN сохраняется в настройках и автоматически восстанавливается при перезапуске сервера.

HTTPS включается автоматически — он необходим для работы Service Worker (кэширование приложения и офлайн-режим) на устройствах, подключённых по сети.

### WAN — доступ через интернет

Два способа (тумблер **WAN** в верхней панели):

- **Cloudflare Tunnel** — выберите режим «Tunnel». Сервер получит публичный HTTPS-адрес вида `https://xxx-xxx.trycloudflare.com`. Требуется установленный `cloudflared`. Адрес меняется при каждом включении
- **Статический IP** — укажите ваш внешний IP и порт вручную. Требуется настройка проброса портов (port forwarding) на роутере. Настройка сохраняется и восстанавливается при перезапуске

### Использование на других устройствах

1. Откройте LAN- или WAN-адрес в браузере устройства
2. Войдите под своим аккаунтом (или попросите администратора создать аккаунт для вас)
3. Интерфейс приложения автоматически закэшируется через Service Worker — при следующих визитах страница загрузится мгновенно, даже если сервер временно недоступен

**Кэширование треков для офлайн-прослушивания:**

- Кнопка ↓ рядом с треком — скачать один трек в кэш браузера
- Кнопка ↓ в шапке списка треков — скачать все треки текущего каталога
- Кнопка ↓ на плейлисте — скачать все треки плейлиста (уже скачанные пропускаются)
- Фильтр «Только кэш» ✓ — показывает только кэшированные треки, счётчик обновляется
- В офлайн-режиме фильтр кэша включается автоматически с уведомлением

Треки хранятся в IndexedDB браузера. Размер кэша отображается в профиле пользователя (иконка пользователя).

## Импорт треков из внешних источников

### VK Music

Для скачивания из VK необходима авторизация:

1. Нажмите «Авторизация VK» в нижней части панели
2. Перейдите по ссылке авторизации — откроется страница VK OAuth
3. После авторизации VK перенаправит на страницу с токеном в URL
4. Скопируйте весь URL из адресной строки и вставьте в поле ввода

**Важно о безопасности VK-токена:**
- Токен хранится **только в оперативной памяти** сервера
- Токен **не сохраняется на диск** и **никуда не передаётся** — при перезапуске сервера авторизацию нужно пройти заново
- Токен используется исключительно для доступа к аудиозаписям через API VK и ни для чего другого

После авторизации вы можете:
- Вставить ссылку на плейлист VK для массового скачивания
- Искать и скачивать отдельные треки через встроенный поиск VK

### Импорт из других источников

Поддерживается импорт плейлистов из Spotify, Apple Music, Яндекс.Музыки и SoundCloud.

| Источник | Как работает |
|----------|-------------|
| **Spotify** | Парсинг списка треков из публичного плейлиста по ссылке |
| **Apple Music** | Парсинг треков через iTunes API |
| **Яндекс.Музыка** | Парсинг через API Яндекс.Музыки |
| **SoundCloud** | Парсинг плейлиста через API SoundCloud |

Импорт работает в два этапа: сначала из внешнего сервиса извлекается список треков (название + исполнитель), затем каждый трек ищется и скачивается через VK Music. Поэтому для импорта из любого источника необходима авторизация VK.

## Организация коллекции

### Нумерация треков

Треки хранятся в формате `0001. Исполнитель - Название.mp3`. Числовой префикс определяет порядок воспроизведения. Ширина нумерации (количество цифр) автоматически подстраивается под общее количество треков.

При скачивании из VK можно выбрать, куда вставить новые треки:
- **Сверху** — новые треки получат номера от 0001, существующие сдвинутся вниз
- **Снизу** — новые треки добавятся после последнего существующего
- **В определённую позицию** — треки вставятся в указанное место, остальные автоматически перенумеруются

### Перемещение треков

- **Drag-and-drop** — включите режим редактирования (кнопка с карандашом в шапке списка), перетащите треки в нужном порядке, нажмите сохранить (✓). Все файлы будут переименованы с новой нумерацией
- **Через редактирование трека** — откройте редактирование (долгое нажатие или правый клик → меню), укажите новый номер в поле «Порядок». Трек переместится на указанную позицию, остальные сдвинутся

При любом перемещении нумерация всех треков в каталоге автоматически пересчитывается для сохранения последовательности без пропусков.

### Переименование и удаление

- **Переименование** — откройте редактирование трека, измените название и/или исполнителя. Файл будет переименован на диске, теги в аудиофайле (ID3, Vorbis и т.д.) обновлены
- **Удаление** — через контекстное меню трека (правый клик или долгое нажатие) → «Удалить». Файл удаляется с диска безвозвратно, требуется подтверждение

### Плейлисты

- **Создать** — вкладка «Плейлисты» → «+ Создать плейлист»
- **Добавить треки** — контекстное меню трека → «Добавить в плейлист» → выбрать нужный плейлист
- **Воспроизвести** — раскройте плейлист нажатием и нажмите «Воспроизвести». Очередь воспроизведения будет построена из треков плейлиста
- **Редактировать** — кнопка с карандашом на плейлисте: изменение названия, удаление треков
- **Удалить** — через редактирование → «Удалить плейлист»
- **Изменить порядок** — перетаскивание плейлистов drag-and-drop

Плейлисты хранятся в файле `.vinyl_playlists.json` внутри каталога и привязаны к папке, а не к пользователю.

## Поиск метаданных

Плеер может автоматически найти и заполнить метаданные (исполнитель, альбом, обложка) для ваших треков.

### Как это работает

1. Откройте каталог с треками
2. Нажмите кнопку поиска метаданных (отображается в интерфейсе при наличии треков без полных метаданных)
3. Плеер проанализирует каждый трек, извлечёт из имени файла исполнителя и название, и отправит запросы к нескольким сервисам одновременно
4. Результаты отображаются как **предложения** — вы просматриваете найденные данные и выбираете, какие изменения применить

### Используемые сервисы

| Сервис | Что ищет | API-ключ |
|--------|----------|----------|
| **Deezer** | Исполнитель, альбом, обложка, дата релиза | Не нужен |
| **iTunes** | Обложка, альбом | Не нужен |
| **Genius** | Обложка (арт трека) | Не нужен |
| **MusicBrainz** | Исполнитель, альбом, обложка через Cover Art Archive | Не нужен |

Все сервисы используются без API-ключей — они предоставляют публичные API.

### Подтверждение изменений

Метаданные **никогда не применяются автоматически**. После завершения поиска вы получаете список предложений, где для каждого трека показаны найденные данные (новый исполнитель, альбом, обложка). Вы отмечаете нужные и нажимаете «Применить». Изменения записываются непосредственно в теги аудиофайла (ID3 для MP3, Vorbis Comment для FLAC/OGG, iTunes atoms для M4A).

Также можно запустить поиск для одного трека — через его редактирование.

## PWA на iPhone/iPad

insideside music можно установить как полноценное веб-приложение на iOS:

1. Откройте адрес плеера в **Safari** (только Safari поддерживает PWA на iOS)
2. Нажмите кнопку **«Поделиться»** (квадрат со стрелкой вверх)
3. Выберите **«На экран Домой»**
4. Приложение появится на рабочем столе как отдельная иконка

### Что даёт PWA-режим

- **Полноэкранный режим** — без адресной строки Safari, приложение выглядит как нативное
- **Кэш приложения** — интерфейс загружается мгновенно через Service Worker, даже без сети
- **Кэш музыки** — закэшированные треки доступны полностью офлайн, хранятся в IndexedDB
- **Управление с экрана блокировки** — play/pause, переключение треков, обложка текущего трека через Media Session API
- **Автоматическое обновление** — при изменении приложения на сервере Service Worker обновляет кэш в фоне

### Особенности iOS

- Service Worker работает только по HTTPS или на localhost — при включении LAN-режима HTTPS активируется автоматически
- При первом запуске PWA может потребоваться восстановление аудиосессии iOS — плеер определяет эту ситуацию и восстанавливает воспроизведение автоматически

## Возможности плеера

### Интерфейс

- **Виниловая пластинка** — анимация вращения с отображением обложки альбома, перетаскивание пластинки для перемотки
- **Адаптивный фон** — цвет фона подстраивается под обложку текущего трека
- **Два режима на мобильных** — переключение между пластинкой и списком треков через тумблер внизу экрана. На вкладке пластинки мобильные кнопки play/next скрываются, так как плеер имеет собственные контролы
- **Три вкладки** — Треки, Альбомы (группировка по альбомам с обложками), Плейлисты
- **Поиск** — мгновенный поиск по названию и исполнителю с фильтрацией в реальном времени
- **Контекстное меню** — правый клик или долгое нажатие на треке: воспроизвести, играть далее, добавить в плейлист, редактировать, удалить, кэшировать

### Воспроизведение

- Play/Pause, предыдущий/следующий трек
- Перемотка по прогресс-бару или вращением пластинки
- Режим перемешивания (shuffle) — перемешивает очередь, сохраняя текущий трек
- «Играть далее» — вставка трека следующим в очередь воспроизведения
- Очередь воспроизведения учитывает текущие фильтры и результаты поиска

### Сброс кэша

Если что-то пошло не так (устаревший кэш, проблемы с Service Worker), перейдите по адресу `/reset` — например, `http://127.0.0.1:7656/reset`. Это очистит все кэши браузера, удалит Service Worker и перенаправит на главную страницу.

## Развёртывание как сервер

### macOS

```bash
pip3 install httpx mutagen vkpymusic musicbrainzngs
python3 vinyl_player.py --public
```

Для автозапуска при входе в систему создайте файл `~/Library/LaunchAgents/com.insideside.music.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.insideside.music</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/bin/python3</string>
        <string>/path/to/vinyl_player.py</string>
        <string>--public</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
</dict>
</plist>
```

```bash
launchctl load ~/Library/LaunchAgents/com.insideside.music.plist
```

При необходимости разрешите входящие подключения в Системные настройки → Брандмауэр.

### Windows

```bash
pip install httpx mutagen vkpymusic musicbrainzngs
python vinyl_player.py --public
```

Для автозапуска при входе в систему:

1. Нажмите `Win+R`, введите `shell:startup`
2. Создайте в открывшейся папке файл `insideside-music.bat`:

Пример systemd-сервиса (`/etc/systemd/system/insideside-music.service`):

```ini
[Unit]
Description=insideside music
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/insideside-music
ExecStart=/usr/bin/python3 vinyl_player.py --public
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable insideside-music
sudo systemctl start insideside-music
sudo ufw allow 7656/tcp
```
